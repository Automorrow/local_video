## 任务清单

### Stream 1: 🔴 Bug 修复（历史删除单条）

#### T1.1 补充 DELETE /api/history/:id 路由

**对应 AC**: AC-016
**对应技术方案**: API 文档 A.3.3
**涉及文件**: `src/modules/api_handler/api_handler.c`
**依赖**: 无

**任务描述**:
在 `api_handler_handle()` 的 DELETE 方法分支中，在现有 `strcmp(path, "history") == 0` 判断之前，增加对 `strncmp(path, "history/", 8) == 0` 的匹配。匹配时从路径中提取 id，调用已有的 `db_manager_history_delete(id)` 函数。

**验证标准 (RED)**:
- [ ] 发送 `DELETE /api/history/1` 返回 200 `{"success":true}`
- [ ] 发送 `DELETE /api/history/abc` 返回 400 错误
- [ ] 发送 `DELETE /api/history/99999`（不存在的 ID）返回 404
- [ ] 发送 `DELETE /api/history` 仍然清空全部（回归）
- [ ] 编译通过，无警告（`-Wall -Wextra -Werror`）

**通俗解释**: 前端点"Remove"按钮发送 `DELETE /api/history/{id}` 但后端只认精确的 `/api/history` 路径（用于清空全部），导致删除单条记录时收到 404。只需加一个前缀匹配分支即可。
