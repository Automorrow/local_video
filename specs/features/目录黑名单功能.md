# 目录黑名单功能 — 功能需求说明书

## 1. 背景与价值

**为什么做**：用户可能有一些不想显示在视频列表中的目录或视频，需要一个方式来屏蔽这些内容，并且可以快速恢复。

**做成什么样**：用户可以通过网页界面管理目录黑名单，被加入黑名单的目录下的视频被标记而不显示，删除黑名单后可立即恢复。

---

## 2. 范围

### 本次做

- 数据库表存储黑名单
- REST API：添加/删除/查询黑名单
- 网页黑名单管理界面
- 视频扫描时跳过黑名单目录
- 添加黑名单时标记视频（不删除），删除黑名单时取消标记并立即扫描恢复
- HTTP 服务器支持 DELETE 方法（用于删除黑名单）
- 缩略图自适应显示（object-fit: cover）
- 前端自动刷新视频列表（添加/删除黑名单后）
- Web 静态文件正确部署到 build/bin/web/static/
- 局域网访问支持（INADDR_ANY）
- 数据库为空时自动扫描
- Toast 通知提示
- 防抖搜索
- 收藏标记
- 继续播放对话框
- 加载动画
- 键盘快捷键
- 历史记录只返回有效的视频（未黑名单且存在）
- 视频播放错误处理

### 本次不做

- 正则表达式匹配 — 后续根据用户反馈再决定
- 通配符匹配 — 后续根据用户反馈再决定
- 批量导入/导出黑名单 — 后续根据用户反馈再决定

## 4. 需求详述

### 4.1 核心流程

1. 用户打开黑名单管理页面
2. 用户输入目录路径并点击"添加黑名单"
3. 系统验证目录路径并添加到数据库
4. 系统将该目录下的已有视频标记为黑名单（不删除）
5. 下次扫描时自动跳过黑名单目录
6. 用户删除黑名单目录时，立即取消标记并扫描恢复视频

### 4.2 交互规则

- 删除操作不需要二次确认（简单快速）
- 表单提交后显示操作结果（成功/失败）
- 输入框失焦时不做实时校验，提交时统一校验
- 历史记录只显示有效的视频
- 从历史记录播放时显示继续播放对话框

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 目录匹配使用前缀匹配，只要视频路径以黑名单目录开头即匹配 |
| BR-002 | 黑名单存储在数据库中，服务器重启后仍然有效 |
| BR-003 | 已在黑名单中的目录不能重复添加 |
| BR-004 | 不存在的目录不能添加到黑名单 |
| BR-005 | 视频使用 blacklisted 标志标记，而非物理删除 |
| BR-006 | 历史记录使用 INNER JOIN 只返回有效的视频 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 添加黑名单目录
  - **Given** 用户在黑名单管理页面
  - **When** 用户输入一个目录路径并点击"添加黑名单"
  - **Then** 该目录被加入黑名单并显示在列表中，视频列表自动刷新

- [ ] **AC-002**: 删除黑名单目录
  - **Given** 黑名单中有目录
  - **When** 用户点击某个目录的"删除"按钮
  - **Then** 该目录从黑名单中移除，视频被取消标记并立即扫描恢复

- [ ] **AC-003**: 扫描时跳过黑名单目录
  - **Given** 某个目录已在黑名单中
  - **When** 系统进行视频扫描
  - **Then** 该目录及其所有子目录下的视频都不会被扫描到数据库中

- [ ] **AC-004**: 添加黑名单时标记视频
  - **Given** 数据库中存在某个黑名单目录下的视频
  - **When** 该目录被添加到黑名单
  - **Then** 该目录下的所有视频被标记为 blacklisted = 1

- [ ] **AC-005**: 正常目录的视频显示
  - **Given** 用户在视频列表页面
  - **When** 视频所在目录不在黑名单中
  - **Then** 该视频正常显示在列表中

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-006**: 添加不存在的目录
  - **Given** 用户输入的目录路径不存在
  - **When** 用户尝试添加到黑名单
  - **Then** 提示错误信息，目录不被添加

- [ ] **AC-007**: 添加重复目录
  - **Given** 用户尝试添加已在黑名单中的目录
  - **When** 点击"添加黑名单"
  - **Then** 提示该目录已存在，不重复添加

- [ ] **AC-008**: 嵌套黑名单目录匹配
  - **Given** 黑名单中有多层嵌套目录（如 /a 和 /a/b）
  - **When** 扫描视频
  - **Then** 只要路径匹配任一黑名单前缀就跳过

- [ ] **AC-009**: 删除黑名单后立即恢复
  - **Given** 用户删除一个黑名单目录
  - **When** 删除操作完成
  - **Then** 该目录下的视频立即被取消标记并扫描恢复

- [ ] **AC-014**: 继续播放对话框
  - **Given** 用户从历史记录播放视频，且上次播放超过 5 秒
  - **When** 用户点击播放
  - **Then** 显示对话框询问是否继续上次位置

- [ ] **AC-015**: 视频播放错误处理
  - **Given** 用户点击播放一个不存在或已黑名单的视频
  - **When** 视频加载失败
  - **Then** 显示友好的 Toast 错误提示

- [ ] **AC-016**: 删除单条历史记录
  - **Given** 历史记录列表中有某条记录
  - **When** 用户点击该记录的"Remove"按钮
  - **Then** 发送 DELETE /api/history/{id} 请求，该条历史记录被删除，列表自动刷新

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-010**: 前缀匹配规则 ← BR-001
  - **Given** 任何目录被加入黑名单
  - **When** 匹配路径
  - **Then** 使用前缀匹配（只要视频路径以黑名单目录开头即匹配）

- [ ] **AC-011**: 黑名单持久化 ← BR-002
  - **Given** 任何情况下
  - **When** 管理黑名单
  - **Then** 黑名单存储在数据库中，服务器重启后仍然有效

- [ ] **AC-012**: 缩略图自适应显示
  - **Given** 视频列表页面显示多个视频卡片
  - **When** 每个视频卡片的缩略图加载完成
  - **Then** 所有缩略图自动填充容器（250px × 150px），保持宽高比不变形

- [ ] **AC-013**: 局域网访问支持
  - **Given** 服务器正在运行
  - **When** 同网段的其他设备访问服务器 IP:端口
  - **Then** 可以正常访问视频列表和播放视频
