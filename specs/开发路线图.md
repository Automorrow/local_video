# 项目开发路线图 (Development Roadmap)

## 0. 策略概览

- **开发策略**: 从基础设施开始，先实现共享模块（地基），然后实现数据库和HTTP服务（核心），接着实现视频扫描和API处理，最后实现前端和增强功能。
- **MVP 目标**: 实现视频扫描、HTTP服务、基础播放功能，完成从0到1的核心闭环。

***

## 1. 里程碑一：项目初始化与基础设施

> 目标：搭建项目骨架，实现共享基础设施模块。

- [ ] **Phase 1.1: 项目初始化**
  - **任务**: 创建目录结构、CMakeLists.txt、公共头文件
  - **说明**: 按照 `specs/项目结构.md` 创建完整的目录树，编写 CMake 构建配置，定义公共类型和宏。
- [ ] **Phase 1.2: 共享基础设施 - 模块化机制**
  - **模块**: `src/shared/module/`
  - **说明**: 实现 `module_init` / `module_run` / `module_exit` 宏，基于 GCC section 属性的模块生命周期管理。
- [ ] **Phase 1.3: 共享基础设施 - 侵入式链表**
  - **模块**: `src/shared/list/`
  - **说明**: 实现内核风格的侵入式链表，包括 `list_add`、`list_del`、`list_for_each`、`container_of` 等宏。
- [ ] **Phase 1.4: 共享基础设施 - 通知链**
  - **模块**: `src/shared/notifier/`
  - **说明**: 基于侵入式链表实现通知链机制，支持优先级注册和事件回调。
- [ ] **Phase 1.5: 共享基础设施 - 日志系统**
  - **模块**: `src/shared/log/`
  - **说明**: 实现简单的日志系统，支持不同日志级别（INFO/WARNING/ERROR）。
- [ ] **Phase 1.6: 共享基础设施 - 线程安全工具**
  - **模块**: `src/shared/thread/`
  - **说明**: 封装 pthread 互斥锁和条件变量，提供便捷的线程安全接口。
- [ ] **Phase 1.7: 共享基础设施 - JSON 库**
  - **模块**: `src/shared/json/`
  - **说明**: 实现轻量级 JSON 生成器（用于 API 响应）。
- [ ] **Phase 1.8: 主入口与配置模块**
  - **模块**: `src/main.c`, `src/modules/config/`
  - **说明**: 实现程序入口，集成模块生命周期机制；实现命令行参数解析。

***

## 2. 里程碑二：MVP (核心闭环)

> 目标：实现最基础的业务流程，完成从 0 到 1 的闭环。

- [ ] **Phase 2.1: 数据库管理模块**
  - **模块**: `src/modules/db_manager/`
  - **说明**: 实现 SQLite3 封装，创建数据库表（视频、历史记录、收藏），提供 CRUD 接口。
- [ ] **Phase 2.2: HTTP 服务模块（基础）**
  - **模块**: `src/modules/http_server/`
  - **说明**: 实现基础 HTTP/1.1 服务器，支持 socket 监听、连接接受、简单请求解析。
- [ ] **Phase 2.3: HTTP 服务模块（静态文件）**
  - **模块**: `src/modules/http_server/`
  - **说明**: 实现静态文件服务，支持 HTML/CSS/JS 文件传输。
- [ ] **Phase 2.4: 视频扫描模块**
  - **模块**: `src/modules/video_scanner/`
  - **说明**: 实现目录递归扫描，识别 MP4/WebM 视频文件，提取元数据存入数据库。
- [ ] **Phase 2.5: API 处理模块（基础）**
  - **模块**: `src/modules/api_handler/`
  - **说明**: 实现 REST API 路由，返回视频列表 JSON 数据。
- [ ] **Phase 2.6: 前端基础页面**
  - **模块**: `web/static/`
  - **说明**: 实现简单的 index.html，显示视频列表，支持点击播放。

***

## 3. 里程碑三：功能完善 (Feature Complete)

> 目标：补充辅助功能，提升用户体验。

- [ ] **Phase 3.1: HTTP 服务模块（视频流式传输）**
  - **模块**: `src/modules/http_server/`
  - **说明**: 实现视频文件流式传输，支持大文件分块读取。
- [ ] **Phase 3.2: HTTP 服务模块（Range Requests）**
  - **模块**: `src/modules/http_server/`
  - **说明**: 实现 HTTP Range Requests 支持，允许视频拖动播放。
- [ ] **Phase 3.3: API 处理模块（搜索与分类）**
  - **模块**: `src/modules/api_handler/`
  - **说明**: 实现视频搜索 API，按目录分类 API。
- [ ] **Phase 3.4: API 处理模块（历史记录）**
  - **模块**: `src/modules/api_handler/`
  - **说明**: 实现播放历史记录的增删查 API。
- [ ] **Phase 3.5: API 处理模块（收藏功能）**
  - **模块**: `src/modules/api_handler/`
  - **说明**: 实现收藏的增删查 API。
- [ ] **Phase 3.6: API 处理模块（随机播放）**
  - **模块**: `src/modules/api_handler/`
  - **说明**: 实现随机视频获取 API。
- [ ] **Phase 3.7: 前端功能增强**
  - **模块**: `web/static/`
  - **说明**: 实现搜索、分类、历史记录、收藏、随机播放按钮和自动下一个功能。
- [ ] **Phase 3.8: 模块间事件集成**
  - **模块**: 各模块
  - **说明**: 集成通知链机制，视频扫描完成、播放开始等事件触发相应处理。

***

## 5. 模块开发顺序速查

### 依赖关系图

```
Phase 1: 基础设施
  ├─ 项目初始化
  ├─ 模块化机制
  ├─ 侵入式链表
  ├─ 通知链
  ├─ 日志系统
  ├─ 线程安全工具
  ├─ JSON 库
  └─ 主入口 + 配置

Phase 2: MVP
  ├─ 数据库管理
  ├─ HTTP 服务（基础）
  ├─ HTTP 服务（静态文件）
  ├─ 视频扫描
  ├─ API 处理（基础）
  └─ 前端基础页面

Phase 3: 功能完善
  ├─ HTTP 服务（视频流式）
  ├─ HTTP 服务（Range Requests）
  ├─ API 处理（搜索/分类）
  ├─ API 处理（历史记录）
  ├─ API 处理（收藏）
  ├─ API 处理（随机播放）
  ├─ 前端功能增强
  └─ 模块间事件集成

Phase 4: 测试与优化
  ├─ 单元测试
  ├─ 集成测试
  ├─ 性能优化
  └─ 文档完善
```

***

**使用指南**:

1. **功能开发**: 按照里程碑顺序，依次开发每个功能模块。
2. 每完成一个模块，请回来勾选进度。
3. 同一个里程碑内的 Phase，可以根据依赖关系适当调整顺序。
