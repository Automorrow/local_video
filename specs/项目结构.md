# 项目结构

## 1. 目录树 (Directory Tree)
```
/
├── specs/                  # 项目核心定义 (产品/技术/结构/规范)
├── docs/                   # 项目文档
├── tools/                  # 辅助工具（不属于主运行时）
├── src/                    # 源代码
│   ├── main.c              # 程序入口
│   ├── modules/            # 业务模块
│   │   ├── http_server/    # HTTP服务模块
│   │   ├── video_scanner/  # 视频扫描模块
│   │   ├── db_manager/     # 数据库管理模块
│   │   ├── api_handler/    # API处理模块
│   │   └── config/         # 配置管理模块
│   ├── shared/             # 共享基础设施
│   │   ├── list/           # 侵入式链表实现
│   │   ├── notifier/       # 通知链实现
│   │   ├── module/         # 模块化生命周期机制
│   │   ├── json/           # JSON解析/生成
│   │   ├── log/            # 日志系统
│   │   └── thread/         # 线程安全工具
│   └── include/            # 公共头文件
│       └── local_video.h   # 项目公共头文件
├── web/                    # 前端资源
│   ├── static/             # 静态资源唯一入口
│   │   ├── index.html
│   │   ├── css/
│   │   └── js/
├── build/                  # 编译输出目录
├── tests/                  # 测试代码
│   ├── unit/               # 单元测试
│   └── integration/        # 集成测试
├── .gitignore
├── BUILD.md                # 构建/发布说明
├── CMakeLists.txt          # CMake构建脚本
├── linker.ld               # 模块自注册链接脚本（Linux）
├── README-WINDOWS.md       # Windows 构建与运行说明
└── README.md
```

## 2. 关键文件说明 (Key Files)
*   **CMakeLists.txt**: 项目构建脚本，定义编译规则和目标。
*   **README.md**: 项目入口文档，包含编译、运行说明。
*   **BUILD.md**: 构建、CI 与发布产物说明。
*   **linker.ld**: Linux 下模块自注册机制依赖的链接脚本。
*   **src/main.c**: 程序入口，负责初始化模块生命周期。
*   **src/include/local_video.h**: 项目公共头文件，定义公共类型和宏。

## 3. 模块说明 (Module Description)
*   **src/modules/**: 业务逻辑的核心模块
    *   `http_server/`: HTTP/1.1服务器实现，支持静态文件和视频流式传输
    *   `video_scanner/`: 视频文件扫描与元数据提取
    *   `db_manager/`: SQLite3数据库管理，视频/历史/收藏数据操作
    *   `api_handler/`: RESTful API处理，JSON响应生成
    *   `config/`: 配置管理，命令行参数解析
*   **src/shared/**: 通用基础设施，不包含具体业务逻辑
    *   `list/`: 内核风格侵入式链表实现
    *   `notifier/`: 通知链机制，模块间事件通讯
    *   `module/`: 模块化生命周期（init/run/exit）实现
    *   `json/`: 轻量级JSON解析/生成库
    *   `log/`: 日志系统
    *   `thread/`: 线程安全工具（互斥锁、条件变量封装）
*   **web/static/**: 前端静态资源唯一入口
    *   `index.html`: 主页面
    *   `css/`: 样式文件
    *   `js/`: 前端脚本

## 4. 文件放置规则 (Placement Rules)
*   **业务模块**: 每个模块独立目录，包含 `.c` 源文件和 `.h` 头文件
*   **模块内拆分**: 当单个模块出现多类职责时，可在模块目录内拆分多个实现文件，并以 `*_internal.h` 承载仅供模块内部复用的声明
*   **公共头文件**: 放在 `src/include/`，供各模块共享
*   **共享基础设施**: 放在 `src/shared/`，与业务逻辑解耦
*   **前端资源**: 静态文件统一放在 `web/static/`，避免与 `web/` 根目录重复维护
*   **测试代码**: 按单元测试/集成测试分类，放在 `tests/` 目录
*   **补充文档**: 构建、平台、模块机制、开发记录统一放在 `docs/` 或顶层入口文档中
*   **辅助脚本**: 与主产品运行时解耦的开发辅助脚本统一放在 `tools/`，不得混入 `src/`、`tests/`、`specs/`
*   **运行产物**: 构建目录、数据库文件、日志、依赖下载目录属于运行或工具链产物，不属于项目源码结构

## 5. 命名约定 (Naming Conventions)
*   **文件名**: 小写字母 + 下划线分隔（如 `http_server.c`）
*   **函数名**: 模块前缀 + 小写字母 + 下划线（如 `http_server_start()`）
*   **类型名**: 使用项目现状中的 `snake_case_t`（如 `lv_error_t`）
*   **宏定义**: 全大写 + 下划线（如 `MODULE_INIT(fn)`）
